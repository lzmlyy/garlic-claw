# 插件开发指南

Garlic Claw 现在使用统一插件运行时：

- `builtin`：内建插件，跟随后端启动，但仍走同一套 manifest、Hook、Host API 和执行协议
- `remote`：远程插件，通过 WebSocket 接入，同样走这套协议

当前不做插件市场，重点是统一协议和统一运行时。

## 概述

插件系统统一了这些概念：

- `manifest`：插件元数据、工具列表、Hook 列表、权限
- `crons`：插件声明或动态注册的定时任务
- `config schema`：插件可声明自己的配置项、默认值和字段类型
- `tools`：可被 AI 或自动化调用的工具
- `hooks`：宿主在关键阶段触发的扩展点
- `routes`：插件可向宿主声明的 HTTP JSON Route
- `host api`：插件访问宿主能力的统一入口
- `transport`：只有传输方式不同，`builtin` 走进程内，`remote` 走 WebSocket

## 远程插件快速开始

### 1. 启动服务

建议先启动开发环境：

```bash
python tools/一键启停脚本.py --build
```

默认插件 WebSocket 地址：

```text
ws://127.0.0.1:23331
```

### 2. 创建插件项目

```bash
mkdir packages/plugins/plugin-my-device
cd packages/plugins/plugin-my-device
npm init -y
```

`package.json`：

```json
{
  "name": "@garlic-claw/plugin-my-device",
  "version": "0.1.0",
  "private": true,
  "main": "dist/index.js",
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js"
  },
  "dependencies": {
    "@garlic-claw/plugin-sdk": "*",
    "@garlic-claw/shared": "*"
  },
  "devDependencies": {
    "typescript": "^5.7.0"
  }
}
```

### 3. 编写插件代码

```typescript
import { DeviceType, PluginClient, type PluginCapability } from '@garlic-claw/plugin-sdk';

const tools: PluginCapability[] = [
  {
    name: 'get_status',
    description: '获取设备状态',
    parameters: {},
  },
  {
    name: 'set_brightness',
    description: '设置亮度',
    parameters: {
      level: {
        type: 'number',
        description: '亮度 0-100',
        required: true,
      },
    },
  },
];

const client = new PluginClient({
  serverUrl: process.env.WS_URL ?? 'ws://127.0.0.1:23331',
  token: process.env.PLUGIN_TOKEN ?? '',
  pluginName: 'my-device-living-room',
  deviceType: DeviceType.IOT,
  manifest: {
    name: 'Living Room Device',
    version: '1.0.0',
    description: '客厅设备插件',
    permissions: [],
    tools,
    hooks: [],
    config: {
      fields: [
        {
          key: 'displayName',
          type: 'string',
          description: '设备在回复中的显示名称',
          defaultValue: '客厅设备',
        },
      ],
    },
  },
});

client.onCommand('get_status', async () => {
  return {
    online: true,
    brightness: 75,
  };
});

client.onCommand('set_brightness', async (params) => {
  const level = params.level as number;
  return {
    success: true,
    brightness: level,
  };
});

client.connect();
```

### 4. 配置环境变量

```bash
export PLUGIN_TOKEN="your-jwt-token"
export WS_URL="ws://127.0.0.1:23331"
```

JWT token 通过 `/api/auth/login` 获取。

### 5. 构建并运行

```bash
npm run build
npm start
```

## Manifest

远程插件最终注册到服务器的 manifest 结构如下：

```typescript
interface PluginManifest {
  id: string;
  name: string;
  version: string;
  runtime: 'builtin' | 'remote';
  description?: string;
  permissions: PluginPermission[];
  tools: PluginCapability[];
  crons?: PluginCronDescriptor[];
  hooks?: PluginHookDescriptor[];
  config?: PluginConfigSchema;
  routes?: PluginRouteDescriptor[];
}
```

其中：

- `id` 由 `pluginName` 决定
- `runtime` 对远程插件固定为 `remote`
- `tools` 是 AI / 自动化可调用的工具
- `hooks` 是宿主扩展点
- `config` 用来声明插件配置项；配置值由宿主持久化，插件通过 Host API 读取
- `routes` 用来声明插件可处理的宿主 HTTP JSON Route

## 工具处理器

`onCommand(name, handler)` 用来注册工具处理器：

```typescript
client.onCommand('set_brightness', async (params, context) => {
  return {
    success: true,
    brightness: params.level,
  };
});
```

处理器签名：

```typescript
(params: JsonObject, context: PluginExecutionContext) => Promise<JsonValue> | JsonValue
```

`context` 中包含：

- `callContext`：当前是谁在调用这个工具，例如聊天工具调用、自动化调用
- `host`：统一 Host API 门面

## Hook 处理器

目前已内置的聊天 Hook 是：

- `chat:before-model`
- `chat:after-model`
- `cron:tick`

注册方式：

```typescript
client.onHook('chat:before-model', async (payload, context) => {
  return {
    action: 'mutate',
    systemPrompt: `${String((payload as {
      request?: { systemPrompt?: string }
    }).request?.systemPrompt ?? '')}\n\n这里可以追加系统提示词`,
  };
});
```

这个 Hook 适合做：

- 对话前上下文增强
- 记忆注入
- 会话级行为修正
- provider/model 路由
- 工具集合裁剪
- 在命中规则时直接 short-circuit

`chat:after-model` 适合做：

- 自动生成会话标题
- 回复完成后的摘要/提取
- 按回复结果触发轻量副作用

`cron:tick` 适合做：

- 周期巡检
- 定时同步
- 缓存清理
- 插件自身心跳或摘要任务

`chat:before-model` 当前输入是：

```typescript
{
  context: PluginCallContext,
  request: {
    providerId: string,
    modelId: string,
    systemPrompt: string,
    messages: PluginLlmMessage[],
    availableTools: PluginAvailableToolSummary[],
    variant?: string,
    providerOptions?: JsonObject,
    headers?: Record<string, string>,
    maxOutputTokens?: number,
  }
}
```

返回值支持三种：

- `action: 'pass'`
- `action: 'mutate'`
- `action: 'short-circuit'`

其中 `mutate` 可以改写：

- `providerId`
- `modelId`
- `systemPrompt`
- `messages`
- `toolNames`
- `variant`
- `providerOptions`
- `headers`
- `maxOutputTokens`

兼容说明：

- 旧版 `appendSystemPrompt` 返回结构仍可继续运行
- 推荐新插件直接使用强版 `pass / mutate / short-circuit`

## Web Route 处理器

插件现在还可以通过统一协议声明宿主内的 HTTP JSON Route。

注册方式：

```typescript
const client = new PluginClient({
  serverUrl: process.env.WS_URL ?? 'ws://127.0.0.1:23331',
  token: process.env.PLUGIN_TOKEN ?? '',
  pluginName: 'my-http-plugin',
  deviceType: DeviceType.API,
  manifest: {
    name: 'My HTTP Plugin',
    version: '1.0.0',
    permissions: ['conversation:read', 'user:read'],
    tools: [],
    hooks: [],
    routes: [
      {
        path: 'inspect/context',
        methods: ['GET'],
        description: '返回当前用户和会话上下文',
      },
    ],
  },
});

client.onRoute('inspect/context', async (request, context) => {
  const plugin = await context.host.getPluginSelf();
  const user = await context.host.getUser();

  return {
    status: 200,
    body: {
      plugin,
      user,
      query: request.query,
    },
  };
});
```

Route 处理器签名：

```typescript
(request: PluginRouteRequest, context: PluginExecutionContext) =>
  Promise<PluginRouteResponse> | PluginRouteResponse
```

当前 Route 入口路径统一为：

```text
/api/plugin-routes/:pluginId/:routePath
```

例如：

```text
GET /api/plugin-routes/builtin.route-inspector/inspect/context?conversationId=xxx
```

这一层目前是：

- JSON Route
- 复用统一插件权限 / Host API / 超时与失败记录
- 由宿主认证用户后再转给插件处理

```typescript
client.onCommand('remember_preference', async (_params, context) => {
  const saved = await context.host.saveMemory({
    content: '用户喜欢咖啡',
    category: 'preference',
  });

  return {
    ok: true,
    saved,
  };
});

client.onHook('chat:before-model', async (payload, context) => {
  const hookPayload = payload as {
    request: {
      providerId: string,
      modelId: string,
      systemPrompt: string,
      messages: Array<{ role: string; content: string }>,
      availableTools: Array<{ name: string }>,
    }
  }
  const config = await context.host.getConfig();
  const memories = await context.host.searchMemories('咖啡', 5);
  const messages = await context.host.listConversationMessages();
  const currentUser = await context.host.getUser();
  const currentPersona = await context.host.getCurrentPersona();
  const personas = await context.host.listPersonas();
  const currentProvider = await context.host.getCurrentProvider();
  const providers = await context.host.listProviders();
  const lastState = await context.host.getState('last-query');
  const cursor = await context.host.getStorage('cursor.last-id');

  await context.host.setState('last-query', '咖啡');
  await context.host.setStorage('cursor.last-id', 'message-42');

  return {
    action: 'mutate',
    systemPrompt: [
      hookPayload.request.systemPrompt,
      `${String((config as { promptPrefix?: string }).promptPrefix ?? '记忆摘要')}：${Array.isArray(memories) ? memories.length : 0} / ${String((currentUser as { username?: string }).username ?? '')} / ${String(cursor ?? '')} / ${String((currentPersona as { personaId?: string }).personaId ?? '')} / ${Array.isArray(personas) ? personas.length : 0} / ${String((currentProvider as { providerId?: string }).providerId ?? '')} / ${Array.isArray(providers) ? providers.length : 0}`,
    ].join('\n\n'),
  };
});

client.onHook('chat:after-model', async (payload, context) => {
  const conversation = await context.host.getConversation();

  if ((conversation as { title?: string }).title !== 'New Chat') {
    return null;
  }

  const generated = await context.host.generateText({
    providerId: (payload as { providerId?: string }).providerId,
    modelId: (payload as { modelId?: string }).modelId,
    system: '你是一个标题生成器，只输出标题本身。',
    prompt: '请为这段对话生成一个简短中文标题',
    maxOutputTokens: 32,
  });

  await context.host.setConversationTitle(
    String((generated as { text?: string }).text ?? '').trim(),
  );

  return null;
});
```

`config.get` 支持两种形式：

- `await context.host.getConfig()`：读取当前插件的完整解析配置
- `await context.host.getConfig('limit')`：读取单个配置项

`storage.*` 用来做**插件私有持久化 KV**：

- `await context.host.getStorage('cursor.last-id')`
- `await context.host.setStorage('cursor.last-id', 'message-42')`
- `await context.host.listStorage('cursor.')`
- `await context.host.deleteStorage('cursor.last-id')`

它和 `state.*` 的区别是：

- `storage.*`：持久化，可跨进程 / 重启恢复
- `state.*`：易失态，只存在当前宿主进程内

`plugin.self.get` 用来读取当前插件自己的摘要信息；`user.get` 用来读取当前调用用户的最小资料。

`llm.generate` / `llm.generate-text` 当前适合做宿主内的小型二次生成，例如：

- 会话标题生成
- 摘要压缩
- 路由/分类判断

`llm.generate` 的优势是可以直接发送多条结构化消息并显式选择 provider/model：

```typescript
const generated = await context.host.generate({
  providerId: 'openai',
  modelId: 'gpt-5.2',
  system: '你是一个严谨的分类器。',
  messages: [
    {
      role: 'user',
      content: '请判断这段话是否是在请求创建自动化',
    },
  ],
  maxOutputTokens: 64,
});
```

`subagent.run` 用来让插件发起一次受控的宿主侧子代理任务。它和直接 `llm.generate` 的区别是：

- 允许宿主子代理在限定步数内使用当前可见工具
- 继续走统一权限、超时、工具过滤和上下文传播
- 默认不会让插件递归调用自己的工具

```typescript
client.onCommand('delegate_summary', async (_params, context) => {
  const result = await context.host.runSubagent({
    providerId: 'openai',
    modelId: 'gpt-5.2-mini',
    messages: [
      {
        role: 'user',
        content: [
          {
            type: 'text',
            text: '请结合当前可用工具，做一个简短总结',
          },
        ],
      },
    ],
    toolNames: ['recall_memory'],
    maxSteps: 4,
  });

  return {
    text: result.text,
    toolCalls: result.toolCalls,
    toolResults: result.toolResults,
    finishReason: result.finishReason,
  };
});
```

`kb.*` 用来让插件在统一协议下读取系统知识库，而不必回退到核心私有逻辑：

```typescript
client.onHook('chat:before-model', async (payload, context) => {
  const hookPayload = payload as {
    request: {
      systemPrompt: string
      messages: Array<{ role: string; content: string | Array<{ type: string; text?: string }> }>
    }
  }
  const latestMessage = hookPayload.request.messages.at(-1)
  const latestUserText =
    typeof latestMessage?.content === 'string'
      ? latestMessage.content
      : ''

  const entries = await context.host.searchKnowledgeBase(latestUserText, 3)
  if (entries.length === 0) {
    return { action: 'pass' }
  }

  return {
    action: 'mutate',
    systemPrompt: [
      hookPayload.request.systemPrompt,
      '与当前问题相关的系统知识：',
      ...entries.map((entry) => `- ${entry.title}: ${entry.excerpt}`),
    ].join('\n'),
  }
})
```

`persona.*` 用来让插件在统一协议下读取和切换当前会话的人设：

```typescript
client.onHook('chat:before-model', async (payload, context) => {
  const hookPayload = payload as {
    request: {
      messages: Array<{ role: string; content: string | Array<{ type: string; text?: string }> }>
    }
  }
  const currentPersona = await context.host.getCurrentPersona();
  const personas = await context.host.listPersonas();
  const writerPersona = await context.host.getPersona('persona-writer');
  const latestMessage = hookPayload.request.messages.at(-1);

  if (
    typeof latestMessage?.content === 'string'
    && latestMessage.content.includes('#writer')
    && currentPersona.personaId !== writerPersona.id
  ) {
    const activated = await context.host.activatePersona(writerPersona.id);

    return {
      action: 'mutate',
      systemPrompt: activated.prompt,
    };
  }

  return {
    action: 'pass',
  };
});
```

`cron.*` 用来让插件在统一协议下声明和管理自己的定时任务：

```typescript
client.onCommand('enable_sync', async (_params, context) => {
  const job = await context.host.registerCron({
    name: 'sync-summary',
    cron: '10m',
    description: '每 10 分钟同步一次摘要',
    data: {
      scope: 'default',
    },
  });

  const jobs = await context.host.listCrons();

  return {
    job,
    count: jobs.length,
  };
});

client.onHook('cron:tick', async (payload, context) => {
  const tick = payload as {
    job: { name: string }
    tickedAt: string
  }

  await context.host.setStorage(`cron.${tick.job.name}.lastTickAt`, tick.tickedAt);
  return null;
});
```

## 配置管理 API

宿主已提供插件治理接口：

- `GET /api/plugins/:name/config`
- `PUT /api/plugins/:name/config`
- `GET /api/plugins/:name/scopes`
- `PUT /api/plugins/:name/scopes`
- `GET /api/plugins/:name/health`
- `GET /api/plugins/:name/events`
- `GET /api/plugins/:name/crons`
- `POST /api/plugins/:name/actions/:action`

当前这些接口走管理员权限，用来管理：

- 插件配置值
- 全局默认启停
- 会话级启停覆盖
- 健康状态与最近错误
- 最近事件日志
- 治理动作

当前已开放的治理动作：

- `health-check`
- `reload`
- `reconnect`

注意：

- 目前仍**不做插件市场 / 包安装 / 软件升级**
- `reload` 对 `builtin` 表示重新装载定义
- `reconnect` 对 `remote` 表示主动断开当前连接，交给插件 SDK 自动重连

## WebSocket 协议摘要

远程插件和服务器之间的核心动作：

| type | action | 方向 | 说明 |
| --- | --- | --- | --- |
| `auth` | `authenticate` | 插件 -> 服务器 | 认证 |
| `auth` | `auth_ok` / `auth_fail` | 服务器 -> 插件 | 认证结果 |
| `plugin` | `register` | 插件 -> 服务器 | 注册 manifest |
| `plugin` | `register_ok` | 服务器 -> 插件 | 注册成功 |
| `command` | `execute` | 服务器 -> 插件 | 调用插件工具 |
| `command` | `execute_result` / `execute_error` | 插件 -> 服务器 | 工具执行结果 |
| `plugin` | `hook_invoke` | 服务器 -> 插件 | 调用插件 Hook |
| `plugin` | `hook_result` / `hook_error` | 插件 -> 服务器 | Hook 返回结果 |
| `plugin` | `route_invoke` | 服务器 -> 插件 | 调用插件 Route |
| `plugin` | `route_result` / `route_error` | 插件 -> 服务器 | Route 返回结果 |
| `plugin` | `host_call` | 插件 -> 服务器 | 调用 Host API |
| `plugin` | `host_result` / `host_error` | 服务器 -> 插件 | Host API 返回结果 |
| `heartbeat` | `ping` / `pong` | 双向 | 心跳保活 |

SDK 已自动处理：

- 认证
- manifest 注册
- WebSocket 收发
- Host API 往返
- 心跳
- 自动重连

## AI 工具命名规则

- 内建插件工具：直接暴露工具名，例如 `save_memory`
- 远程插件工具：自动加插件前缀，例如 `pc-my-host__list_directory`

这样内建插件可以保持稳定的人类友好工具名，远程插件则天然避免冲突。
