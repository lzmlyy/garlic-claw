# 扩展内核契约说明

本文档定义 Garlic Claw 当前已经落地、并且后续继续按此维护的 extension kernel contract。

目标只有一个：

- 内核像 C 一样简单，只保留稳定边界与少量原语
- SDK 像 C++ 一样提供作者体验层，但不把糖衣反向固化进 core

## 1. 统一发生在哪里

统一只发生在 runtime contract，不发生在 authoring format。

也就是说：

- `plugin` 继续是宿主原生扩展协议
- `MCP` 继续是外部能力/资源协议，通过 adapter 接到 kernel
- `skill` 继续兼容现有 skill 生态，不强制改写成 plugin 包

当前仓库里的对应落点：

- `plugin`
  - `packages/shared/src/types/plugin.ts`
  - `packages/server/src/plugin/plugin.module.ts`
  - `packages/server/src/plugin/bootstrap/plugin-bootstrap.service.ts`
  - `packages/plugin-sdk/src/authoring/index.ts`
  - `packages/plugin-sdk/src/client/index.ts`
  - `packages/plugin-sdk/src/host/index.ts`
  - `packages/plugin-sdk/src/index.ts`（兼容导出面）
- `MCP`
  - `packages/server/src/execution/mcp/mcp.service.ts`
  - `packages/server/src/execution/tool/tool-registry.service.ts`
- `skill`
  - `packages/server/src/execution/skill/skill-session.service.ts`
  - `packages/server/src/execution/skill/skill-registry.service.ts`
  - `packages/server/src/execution/tool/tool-registry.service.ts`

作者写什么格式，由各自生态决定；运行时只看它们最终暴露出来的 capability surface。

## 2. 内核一等原语

kernel 只保留两类一等原语：

- `action call`
- `event subscription`

当前代码中的对应关系：

- `action call`
  - plugin tools
  - MCP tools
  - active skill package tools
  - host api methods
- `event subscription`
  - plugin hooks
  - chat / message / automation / tool / response / lifecycle hook 链

有些东西看起来不是这两类，但本质上只是边缘入口：

- `route`
  - 是 HTTP ingress，不是扩展之间的第三套核心原语
- `cron`
  - 是 host 调度入口，不是扩展直连协议
- `subagent`
  - 是受控 workflow delegation，仍通过 host action 发起，并继续走 tool / hook / permission

## 3. exported surface

默认私有，只允许显式导出的 surface 被发现或调用。

当前已经执行的规则：

- plugin 只有在 manifest 中声明的 `tools / hooks / routes / crons / config` 才进入 runtime
- MCP 只有配置里启用、并成功接入的 server/tool 才进入统一工具层
- skill 只有会话显式激活后，才会进入：
  - prompt/context 组装
  - tool allow/deny 解析
  - skill asset/script tool 暴露

因此：

- capability 发现默认只看 exported 面
- skill 解析默认只看 active skill 的 exported 面
- 扩展之间不允许裸对象直连

内核内部允许做 fast-path，例如：

- builtin transport 进程内调用
- runtime 内部 lazy resolver
- tool provider 对本地状态做单次快照复用

但这些只是实现优化，不构成作者可依赖的新语义。

## 4. Skill 的双面角色

skill 不再被当成第三套厚 runtime。

当前 skill 的两面分别是：

- 对 AI：
  - `prompt`
  - `policy`
  - `constraint`
  - `context assembly`
  - 主要落在 `SkillSessionService`
- 对 runtime：
  - `active skill selection`
  - `tool allow/deny`
  - `asset/script package`
  - `skill package tool surface`
  - 主要落在 `SkillSessionService`
  - skill 触发的子代理 / workflow delegation 最终通过 host runtime 进入 `RuntimeHostSubagentRunnerService`

skill 的依赖解析默认继续走 capability contract，而不是写死某个扩展名：

- tool allow/deny 先看统一工具名
- skill package 额外资产通过 `skill__*` 工具暴露
- 是否允许读资产、跑脚本，取决于 trust level 与 exported asset

如果后续需要为特定生态补覆盖层，应该做成 override，而不是把该生态写成默认绑定方式。

## 5. 最小状态原语

kernel 默认只保留少量通用状态原语，不继续为了每种扩展能力新增专用核心面。

当前保留并已经可用的原语：

- `private scoped KV`
  - `storage.*`
  - 作用域：
    - `plugin`
    - `conversation`
    - `user`
- `ephemeral scoped state`
  - `state.*`
  - 作用域：
    - `plugin`
    - `conversation`
    - `user`
- `exported resource snapshot`
  - `provider.*`
  - `persona.*`
  - `kb.*`
  - `conversation.get`
  - `conversation.messages.list`
  - `user.get`
  - `plugin.self.get`
- `append-only event log`
  - `log.list`
  - `log.write`
  - `PluginPersistenceService.recordPluginEvent(...)`
  - `PluginPersistenceService.listPluginEvents(...)`

这一层的约束是：

- 优先复用统一原语表达扩展状态
- 不再默认为扩展新增专用 Prisma schema
- 只有宿主业务本体确实需要的一等实体，才留在 core schema 中

## 6. builtin 的角色

builtin 是参考实现，不是长期特权实现。

当前已经按这个方向收口：

- `builtin` 和 `remote` 共用同一套 plugin manifest / permission / hook / route / host api / runtime 语义
- 大量原本在 core 里的能力已经迁为 builtin consumer：
  - memory
  - kb context
  - conversation title
  - automation tools
  - persona router
  - subagent delegate
  - governance / response / tool / lifecycle recorder

后续原则：

- 新能力先判断能否做成 plugin / MCP / skill / tool
- builtin 优先作为“怎么消费内核”的样例
- 不再让 builtin 成为继续堆 core 特判的借口

## 7. 维护规则

新增能力时按下面顺序判断：

1. 能否只是一个新的 plugin / MCP / skill adapter？
2. 能否只是给现有 action/event 增加 surface，而不是再造原语？
3. 能否只靠 `storage / state / exported snapshot / event log` 表达状态？
4. 只有前三步都不成立，才考虑进入 core。

不应再做的事：

- 再造第三套扩展 runtime
- 让 skill 反向绑死某个插件格式
- 让 MCP 绕开统一工具层直连聊天主链
- 为单个 builtin 添加新的 core 私有入口
- 把实现层 fast-path 暴露成作者语义
